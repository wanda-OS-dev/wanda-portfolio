## 2024-03-19 - [Avoid Inline Array Methods with Framer Motion and Static Data]
**Learning:** This codebase uses static data objects containing arrays (like tags) alongside Framer Motion animations across Next.js app router pages. Using inline slice methods (`project.tags.slice(0, 4)`) within lists mapped directly to `motion.div` elements caused the array to be recreated during every re-render. This triggered garbage collection and resulted in subtle micro-stutters during component enter animations.
**Action:** When rendering lists wrapped in Framer Motion where partial data subsets (like truncated arrays) are needed, always use `useMemo` to pre-calculate and cache the transformed data structure once per mount instead of doing inline transformations inside the map function.
