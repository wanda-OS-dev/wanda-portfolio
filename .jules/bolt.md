## 2025-05-14 - Inline array operations during component renders
**Learning:** In React components that combine static data arrays with mapping over arrays in render, inline operations such as `Array.prototype.slice(0, 4)` executed inside a `.map()` closure create unnecessary Garbage Collection (GC) pressure upon every re-render. When mapping over constants, or properties that are derived from constants, mapping them outside the component avoids putting that burden inside the React component's lifecycle entirely. Even with libraries like Framer Motion (which often bypasses React render loops for animations but does re-render components on route changes or specific hooks), preventing inline allocations inside map loops keeps performance stable.
**Action:** Always pre-calculate derivative lists (e.g., truncated tags or styled object references) and styles either once per mount using `useMemo` (if dependent on component state/props) or at module level (if derived from static imports) to maintain a stable framerate and reduce micro-stutters during re-renders.
