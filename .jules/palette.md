## 2026-03-22 - Dynamic Form State Accessibility
**Learning:** React state changes (like form success/error messages) are entirely invisible to screen readers unless explicitly marked. While HTML5 `required` attributes handle standard validation gracefully, custom async form flows require `aria-live` regions (`polite` for success, `assertive` for errors) and explicit `aria-busy` indicators on submit buttons so users aren't left wondering if their submission worked. Also, providing visual cues for required fields alongside semantic HTML5 properties ensures both visual and screen-reader users have parity in expectations.
**Action:** Always wrap dynamic success messages in `aria-live="polite"`, use `role="alert"` + `aria-live="assertive"` for async form errors, add `aria-busy` to loading buttons, and augment HTML5 `required` fields with `aria-hidden="true"` visual indicators.
