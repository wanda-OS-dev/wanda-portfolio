## 2025-03-13 - [Unvalidated API Input & HTML Injection in Outbound Emails]
**Vulnerability:** The `/api/contact` endpoint accepted unbounded input sizes and potentially non-string objects for its body payload. Additionally, it directly embedded these inputs into HTML templates sent via `nodemailer`, risking HTML injection and excessive memory utilization (DoS). Client IP was also naively extracted, allowing simple spoofing.
**Learning:** Without explicit type-checking, length limits, and HTML escaping, user-submitted data can compromise the integrity of outbound emails and potentially crash Next.js API routes due to unexpected data types or memory exhaustion.
**Prevention:** Always validate payload structures (`typeof === 'string'`) and bounds (length limits) *before* processing. Always sanitize (escape HTML entities) user inputs before rendering them into HTML strings or templates. Prefer standard `req.ip` for IP addresses before falling back to `x-forwarded-for` and parse headers appropriately.
