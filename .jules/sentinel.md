## 2025-03-20 - Fix HTML Injection in Contact form API
**Vulnerability:** Contact form API route (`src/app/api/contact/route.ts`) accepted raw user input (`name`, `email`, `message`) and concatenated them directly into the HTML body of an email sent by `nodemailer`. This is a classic HTML Injection vulnerability allowing an attacker to insert arbitrary HTML (phishing links, tracking pixels). The route also lacked type checking and length validation, which could be abused to cause server crashes or Denial of Service (DoS) by sending enormous payloads or unexpected types.
**Learning:** `nodemailer`'s `html` field does not automatically sanitize input. In Next.js App Router API routes receiving JSON payloads, inputs must be explicitly type-checked, length-validated, and HTML-escaped before being embedded into HTML templates.
**Prevention:** Always validate input types (`typeof === 'string'`), enforce reasonable length constraints (`name.length > 100`), and apply HTML escaping (e.g., replacing `&`, `<`, `>`, `"`, `'`) for any user-provided data interpolated into an HTML context.
